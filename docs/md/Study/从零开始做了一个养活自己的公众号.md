# 从零开始做了一个养活自己的公众号

先来一个自我介绍，大家好，我是程序员吴师兄。2018 年底开始运营程序员技术公众号，一年时间涨粉 13w；开源项目 LeetCodeAnimation 作者，GitHub 62k star，全球排名 55。

![image.png](https://raw.githubusercontent.com/zhengstar94/zhengstar94.github.io/main/docs/assets/img/2022/06/10.png)

马上进入正题，本次的分享围绕技术公众号，侧重于个人的经历与一些实操的细节，希望能带给你一些思考。

2018 年由于某些原因一直摸索着搞一份副业，做过家教、接过外包、闲鱼上卖过虚拟资源，倒腾了十个月才挣了一两千，在这十个月的过程中，基本上每天都会去研究怎么才能搞好副业，虽然没挣什么钱，但认知还是发生了改变，再也不是之前那种程序员思维，以为技术就是一切，提升技术就够了。



当时我比较崇拜的一个技术大佬发表了一篇博客，说他在一个平台写了一些付费文章，文章的内容恰巧是我想知道的技术点，于是我前往付费订阅。

隔几天发现有几百人订阅了，算一下收入大概有好几千，由于之前一直在琢磨着挣钱，下意识地产生了想法：我能不能也写一些付费文章？

**写什么内容呢？**



思考了一下我能写的话题，最后决定写算法相关的。



当时，由于我是非科班程序员，数据结构和算法一窍不通，但为了进大厂，被迫去学习，下面这些视频看了几百个小时，因为很有趣，有很多动画帮助理解。

![image.png](https://raw.githubusercontent.com/zhengstar94/zhengstar94.github.io/main/docs/assets/img/2022/06/11.png)

于是，我打算模仿这个视频写一些算法文章，即用 PPT 动画的形式讲解 LeetCode 的题目，核心点是使用 PPT 的动画来呈现解题的思路，帮助新手更好的理解。

- 会使用 PPT 的动画
- 会解 LeetCode 的题
- 想挣钱



这三个因素让我把每天的业余时间都花在解题、做动画、写文章上，幻想着它能火能挣钱。



![image.png](https://raw.githubusercontent.com/zhengstar94/zhengstar94.github.io/main/docs/assets/img/2022/06/12.png)

![image.png](https://raw.githubusercontent.com/zhengstar94/zhengstar94.github.io/main/docs/assets/img/2022/06/13.png)

![image.png](https://raw.githubusercontent.com/zhengstar94/zhengstar94.github.io/main/docs/assets/img/2022/06/14.png)

![image.png](https://raw.githubusercontent.com/zhengstar94/zhengstar94.github.io/main/docs/assets/img/2022/06/15.png)

在平台上更新了好几天，大概是免费的缘故，时不时有人来订阅，我当时想着等免费人数破 100 就转为付费模式，没想到幸福来得很突然，由于当时平台上的内容比较少，我又更新的很频繁且免费，内容还与众不同，于是小编在它的官方服务号进行了一次推送。

![image.png](https://raw.githubusercontent.com/zhengstar94/zhengstar94.github.io/main/docs/assets/img/2022/06/16.png)

推送的时候我在散步，不知道小编在推荐，手机一下子收到很多信息，xx、xx、xx订阅了你的专栏，短短一个小时就有了 2000 多的订阅，我赶紧跑回家设置为付费模式，一个晚上挣了小几百。

![image.png](https://raw.githubusercontent.com/zhengstar94/zhengstar94.github.io/main/docs/assets/img/2022/06/17.png)

提现那一刻的心情应该和生财有术的朋友们挣到第一笔副业收入的心情是一样的：仿佛打开了新世界，自己有无限可能。



![image.png](https://raw.githubusercontent.com/zhengstar94/zhengstar94.github.io/main/docs/assets/img/2022/06/18.png)



由于我经常在知乎搜索副业相关的话题，所以信息流基本上都是和副业有关的回答，有一次在知乎上看到一个程序员在写公众号，说他有 3w 粉丝，靠接广告和社群一年挣了 7w，我关注了他的公众号，发现内容一般，感觉我的内容比他更好，我搞公众号会不会也有搞头？一年挣 3w 就满足了。



于是，我开始把我的文章搬运到公众号上，内容有了剩下的就是引流了：我的内容是优质的，它值得被更多人看到。

![image.png](https://raw.githubusercontent.com/zhengstar94/zhengstar94.github.io/main/docs/assets/img/2022/06/19.png)

花了一周的时间把知乎上关于公众号引流的方法都看了一遍同时实操：

豆瓣可以引流？行，我去豆瓣发文章！



即刻可以引流？行，我去发状态！

贴吧可以引流？行，我去发帖！



V2ex、CSDN、segmentfault、简书、博客园、掘金、GitChat、知乎专栏、B站专栏，只要能发文章的地方我都去发。

当时模仿了某个大 V 的结尾来给公众号引流，由于知乎权重低，马上被禁言了。

![image.png](https://raw.githubusercontent.com/zhengstar94/zhengstar94.github.io/main/docs/assets/img/2022/06/20.png)

一个月下来，粉丝数到了 1800，感觉这样涨粉很费力。

![image.png](https://raw.githubusercontent.com/zhengstar94/zhengstar94.github.io/main/docs/assets/img/2022/06/21.png)

所以，我在琢磨着有没有更高效又安全的引流方法。

前面也提到过，我是一个程序员，热衷于提升技术，有一个习惯，每天上班的时候都喜欢打开 GitHub Trending 瞅一瞅看看有没有什么好玩的仓库。

2017 年的时候开源了一个仓库上过几天热榜。

（题外话：2019 年发现有人用我的这个仓库的源码做了一个 App 上架）

如果我搞一个仓库，里面存放我的文章，在文章里面引流到公众号，如果仓库火了那流量自然而然的有了。



立马创建了一个仓库：LeetCodeAnimation，把公众号的文章搬运过去，同时在 V2ex 进行了一波宣传。

![image.png](https://raw.githubusercontent.com/zhengstar94/zhengstar94.github.io/main/docs/assets/img/2022/06/22.png)

或者是运气，没几天就在微博上被几个大 V 比如「爱可可-爱生活、GitHubDaily」推荐，当天 star 数千，第二天上了 Trending 版，后面连续好几周占据 GitHub Trending 榜首的位置，个人排名甚至超过了 Microsoft、Google。

![image.png](https://raw.githubusercontent.com/zhengstar94/zhengstar94.github.io/main/docs/assets/img/2022/06/23.png)

基于这个仓库，我走上了开挂的道路，基本上每天都有好几百的关注，并且都是精准粉丝，这也导致文章打开率很高，维持在 15% 左右，3w 粉丝的时候阅读量经常是四五千，通过这个仓库大概引流了五六万的粉丝。



从此，我在其它平台不宣传公众号而是宣传我的 GitHub 仓库，一方面避免了被平台禁言删帖，并且经过二次筛选，吸引的都是精准用户，另外一方面又可以增加点赞量更加长时间的待在 Trending 版上吸引更多的其它的程序员。



在这里提出一个我的想法，很多地方都是禁止直接留公众号引流，但对于公众号文章链接的规定没有那么严，可以把引流信息放在文章里，然后在其它平台留下引流的文章链接，转化率没有直接留那么高，但胜在安全和长久。



我把公众号的文章进行整合，发布到知乎。

![image.png](https://raw.githubusercontent.com/zhengstar94/zhengstar94.github.io/main/docs/assets/img/2022/06/24.png)

其中一个热门回答获得了一百万阅读，通过这个回答也基本上每天有几十个流量关注。



时间来到了 2019 年 4 月份，认识了很多技术号号主，比如帅地、武哥、程序员小灰、Python爱好者社区的勇哥等，算是进入了一个圈子，也了解到更多的玩法：互推、单推、联合送书、联合抽奖等。



基本上这些玩法都要保证阅读量，很考验一个号的粘性，我的号当时粘性还不错，因此虽然只有三四万粉却可以和很多十几万的号互推，一来二回涨了不少，行情好的时候一天涨两三千。



在这个过程中也通过给大号比如 CSDN、AI科技等投稿涨了不少粉，当时投稿的人较少，所以采纳几率很高。

![image.png](https://raw.githubusercontent.com/zhengstar94/zhengstar94.github.io/main/docs/assets/img/2022/06/25.png)

现在很考验技巧了，我观察过 2020 年开始搞技术公众号的人，发现发展的比较好的都是那些善于利他的人，他们的投稿方式让人很舒服。



来看两种投稿方式，如果你是小编，你会倾向于采纳谁的投稿？



1、第一种投稿



吴师兄，您好，我是公众号「xxx」的号主，想给您的公众号「xxx」投稿一篇文章。



文章标题：xxx



文章链接：xxx



文章亮点：



1、文章内容亮点xxx

2、有xx号和xx号转载过，打开率都在 20% 以上，留言和在看数据都不错，这些号的属性和您的公众号相符，如果您转载也能产生不错的效果。



所有文章均已开长白，欢迎转载，如果有什么要求欢迎联系我：）

2、第二种投稿



给所有号主群发文章，称呼都不带，然后逼迫式的追问。

![image.png](https://raw.githubusercontent.com/zhengstar94/zhengstar94.github.io/main/docs/assets/img/2022/06/26.png)

答案自然不言而喻，第一种方式可以让小编第一时间明白文章是否符合要求，同时出现过好几次准备转载第二种投稿方式的文章，结果发现对方没有开白，也就懒得再去转了。

并且，那些发展很好的号主都很活跃，会给你的广告文留言点赞，会在你的干货文提出思考，会在你的水文骚话互动，反正每天你都能在你的文章里面看到他们的身影，那对于他们的投稿肯定是更加乐于接受的。

每年都有人问现在搞公众号是否来的及，尽管很多人唱衰，但每年都有很多人做起来了，其它领域我不了解，但 2019、2020 年写技术公众号发展起来的人不在少数，因此我对 2021 年开始写技术公众号的人起乐观态度：



- 技术号坑都被踩完了，安安心心写原创文即可
- 已经有技术号联盟，投稿转载越来越规范，好的内容能够迅速传播涨粉
- 变现的方向比较明朗，如果能做到 3w 粉，一个月妥妥的可以挣个不少于工资的收入



比如我聊的来的几个号主「编程指北」、「小林coding」、「 编程技术宇宙」、「四猿外」，他们都 2020 年才开始写公众号，但都靠爆款做起来的，他们轻易不写文，但写的每一篇文章都是精品，很多号主闻风而动申请转载，写一篇涨 1000 粉，写十篇涨 10000 粉，直接冷启动起飞。

![image.png](https://raw.githubusercontent.com/zhengstar94/zhengstar94.github.io/main/docs/assets/img/2022/06/27.png)

他们都是通过单篇爆款文章的机会，迅速累加优势，短时间内进入更大的资源圈子，度过了写公众号的阶段，后期只需要不断的长期输出优质的内容做公众号即可。

上面这段话你可能不太容易理解，我举个例子吧。

比如写一篇内容为 HashMap 源码分析的文章，这种文章很难写出爆款，无法通过内容涨粉，也就是说很难度过写公众号的阶段，但这类文章也是属于优质的文章，在做公众号阶段是十分有利的，对读者有帮助，读者也爱看，读者也认可。

相反的，如果你写一篇内容为计算机专业学生该如何提高自己类似的文章，读者收藏转发的动机会更强烈，其它号主主动联系转载的机会也会很大，这样一篇爆款文，可以极大的吸引一些你的铁杆粉丝，后面你再写一些深奥的比如红黑树的删除，也是会追着看。

优质的内容任何时候都有市场，不管是 2014 还是 2020 还是 2021。
2015 年的时候我在一家初创公司负责技术开发，当时公司的主营业务就是微博和公众号，但是我当时沉迷于技术提升，只想通过优化代码来给用户更好的体验，对于公司的发展漠不关心，也就没有进一步去了解公众号的情况，直到 2019 年我搞起公众号后才知道自己错过了多大的机会，很多时候认知跟不上，机会摆在眼前都没用。

（目前这家公司依托于公众号已经发展成了几百人的大团队）

如果你想发展一门副业，可以考虑先从公众号入手，公众号是最低成本的创业机会，一个邮箱一个身份证就能开搞，涵盖了内容产出、营销、变现等商业思维，对于改变一个人的固有认知有很大的帮助，不再只知道上班挣钱。
