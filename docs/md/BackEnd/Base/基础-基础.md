
# 基础

## 重载重写区别
### 重载
 > 重载就是同样的方法根据输入数据的不同，做出不同的处理。
发生在同一个类，方法名必须相同，参数类型不同，个数不同/顺序不同，方法访问值和访问修饰符可以不同。
注意：不能有方法名字相同、参数类型相同却返回不同类型的方法。

> 重载就是同一个类中多个桶面方法根据不同的传参来执行不同的逻辑处理

### 重写
> 重写发生在运行期，是子类对父类的允许访问的方法的实现过程进行重新编写
> 1.返回值类型、方法名、参数列表必须相同，抛出的异常范围小雨等于父类，访问修饰符范围大于等于父类
> 2.如果父类方法访问修饰符为```private/final/static``` 则子类不能重写该方法，但是被static修饰的方法能够被再次声明。
> 3.构造方法无法被重写

> 重写就是子类对弗雷方法的重写构造，外部样子不能改变，内部逻辑可以改变

方法重写要遵循“两同两小一大”
- “两同”即方法名相同，形参列表相同
- “两小”子类抛出的异常/返回值类型 小于等于父类
- “一大” 子类方法的访问权限必父类访问权限更大或相等

#### 总结

|区别|重载|重写|
|-------|-------|-------|
|发生范围|同一个方法|子类|
|参数列表|必须修改|一定不能修改|
|返回类型|可修改|子类方法返回值类型**小于等于**父类方法返回值类型|
|异常|可修改|子类方法抛出异常**小于等于**父类方法抛出类型|
|访问修饰符|可以修改|可以降低限制，不能做更严格的修改|
|发生阶段|编译期|运行期间|


## 三大特性：封装、继承、多态

### 封装
封装把一个对象的属性私有化，同时提供一些可以被外界访问的属性的方法，如果属性不想被外界访问，我们不必提供方法给外界访问

### 继承
继承是使用已存在的类的定义作为基础建立新类的技术，新类的定义可以增加新的数据或新的功能，也可以用父类的功能

**关于继承以下3点**
1. 子类拥有父类对象的所有属性和方法（包括私有），但是父类的私有属性和方法子类无法访问，只是拥有。
2. 子类可以拥有自己的属性和方法，可以对父类进行扩展。
3. 子类可以用自己的方式实现父类的方法。

### 多态
一个引用变量指向哪个类的实例对象，该引用变量发出的方法调用到底是哪个类中实现的方法，必须由程序运行期间菜呢个决定。

实现方法：继承（多个子类对同一方法的重写）和接口（实现接口并覆盖同一个方法）

## 接口与抽象类的区别
1. 接口的方法默认是public，所有方法在接口中不能有实现，抽象类可以有非抽象方法；
2. 接口中除了static、final变量，不能有其他变量，抽象类则不一定；
3. 一个类可以实现多个接口，但只能实现一个抽象类。接口自己本身可以通过extends关键字扩展多个接口；
4. 接口的方法默认是public，抽象方法可以有public、protected和default这些修饰符(抽象方法就是为了被重写所以不能使用private关键字)
5. 抽象是对类的抽象，是一种模版设计，而接口是对行为的抽象，是一种行为的规范

## 抽象类必须有抽象方法吗
抽象类不一定包含抽象方法，但是包含抽象方法的类一定要被声明为抽象类

## 抽象类能使用final关键字吗
抽象类不能用final修饰，当final修饰一个类说明不能被继承，final类所有的成员方法都被隐式指定为final方法，违背抽象类存在的意义。

## String的创建方式
```java
String str="abdc" 与 String str1=new String("abcd")
```
- 第一种方式先检测字符串常量池有没有abcd，如果字符常量池没有，则创建一个，然后str指向字符串常量池的对象；如果有，则直接将str指向字符串常量池中的“abcd”
- 第二种方式是直接在堆内存空间创建一个新的对象
推荐使用第一种方式创建字符串

str与str1不等，因为一个是堆内存中的String对象一个是常量池中的String对象。

## String StringBuffer StringBuilder区别？String为什么不可变？

String类使用final关键字修饰集成字符数组来保存字符串，所以String不可变

StringBuilder与StringBuffer没有final关键字修饰，所以两种对象都是可变的；这2个类都是继承自AbstractStringBuilder

- 线程安全性
String对象不可变，所以可以理解为常量，线程安全
StringBuffer对方法加了同步锁所以是线程安全的
StringBuilder没有对方法加了同步锁所以是非线程安全的

- 性能
每次对String类型进行改变的时候，都会生成一个新的String对象，然后将指针指向新的String对象。StringBuffer对象本身进行操作，而不是生成新的对象并改变对象的引用。相同情况下使用 StringBuilder 相比使用 StringBuffer 仅能获得 10%~15% 左右的性能提升，但却要冒多线程不安全的风险。

### 总结
1. 操作少量的数据：使用String
2. 单线程操作字符串缓冲区下操作大量数据：使用StringBuilder
3. 多线程操作字符串缓冲区下操作大量数据：使用StringBuffer

### 常见面试题
```java
String str_1 = new String("ab"); 
String str_2 = new String("ab"); 
String str_3 = "ab";

System.out.println(str_1 == str_2); 
System.out.println(str_1 == str_2.intern()); System.out.println(str_1.intern() == str_2.intern()); System.out.println(str_1 == str_3); 
System.out.println(str_1.intern() == str_3);
```
**输出结果：**
```java
false
false
true
false
true
```

**解释：**
> 1. ==，基数数据类型对比的是值，引用类型对比的是地址。另外，equal对比的是哈希值<br>	
> 2. 两个new出来的对象，地址肯定不同，所以是false
> 3. intern()，直接把值推进了常量池，所以两个对象都做了intern()操作，对比的是常量池的值<br>
> 4. ```str_3="ab"```，赋值，JVM编译器做了优化，不会重新创建对象，直接引用常量池里的值，。所以```str_1.intern() == str_3```对比结果是true




## ==与equal
**==**：作用是判断两个对象的地址是不是相等。即，判断两个对象是不是同一个对象(基本数据类型的==比较的是值，引用数据类型的==比较的是内存地址)

**equais()**；作用是判断两个对象是否相等，但一般有两种使用情况
1. 类没有覆盖equals()方法。通过equals()比较该类的两个对象时，等价于通过“==”比较这两个对象。
2. 类覆盖equals()方法。一般，我们都覆盖equals()来判断两个对象内容是否相等，若想等，则返回true（即认为这两个对象相等）

```java

public class Test1 
{
    public static void main(String[] args) {
        String a = new String("ab"); // a 为一个引用
        String b = new String("ab"); // b为另一个引用,对象的内容一样
        String aa = "ab"; // 放在常量池中
        String bb = "ab"; // 从常量池中查找
        if (aa == bb) // true
            System.out.println("aa==bb");
        if (a == b) // false，非同一对象
            System.out.println("a==b");
        if (a.equals(b)) // true
            System.out.println("aEQb");
        if (42 == 42.0) { // true
            System.out.println("true");
        }
    }
}

```


**说明**
- String中的equals方法是被重写过的，因为object的equals方法是比较的对象的内存地址，而String的equals方法比较的是对象的值；
- 当创建String类型的对象时，虚拟机会在常量池查找有没有已经存在的值和要创建的值相同的对象，如果有就把它赋给当前引用。如果没有就在常量池重新创建一个String对象。


## hashCode与equal
> 重写过hashcode和equals么，为什么重写equals必须重写hashcode方法

### hashCode()介绍
hashCode()的作用是获取哈希码，也称为散列码；hashCode()定义在Obejct.java中，意味着java中任何类都包含有hashCode()函数。它存储的是键值对，特点是能根据key快速检索出对应的value

### 为什么要有hashCode
hashCode的作用就是获取哈希码，哈希码的作用就是确定对象在哈希表表中的索引位置；hashcode()作用就是获取对象的散列码，进而确定该对象在散列表中的位置。

### hashCode()与equals相关规定
1. 如果两个对象相等，则他们的hashCode一定相同；
2. 两个对象相等，对两个对象分别调用equals方法都返回true；
3. 两个对象有相同的hashcode值，他们也不一定是相等的(不同的对象可能产生相同的hashCode)
4. equals方法被覆盖过，则hashCode方法也必须覆盖
5. hashCode()的默认行为是对堆上的对象产生独特值。如果没有重写hashCode()，则该class对象无法相等

## 反射机制与应用场景

### 反射机制介绍
java反射机制是在运行状态中，对任意一个类，都能够知道这个类的所有属性和方法；对于任意一个对象，都能够调用它的任意一个方法和属性；这种动态获取的信息以及动态调用对象的方法的功能成为java语言的反射机制。

### 静态编译与动态编译
- 静态编译；在编译时确定类型，绑定对象
- 动态编译：在运行时确定类型，绑定对象

### 反射机制优缺点
- 优点：运行期类型的判断，动态加载类，提高diamanté灵活度。
- 缺点：性能瓶颈：反射相当于一系列解释操作，通知JVM要做的事情，性能比直接的java代码慢

### 应用场景

1. JDBC连接数据库使用Class.forName()通过反射加载数据库的驱动程序
2. Spring框架也用到很多反射机制，最经典就是xml配置模式，通过xml配置装载bean的过程
	1. 将程序内所有xml或properties加载到内存中；
	2. java类解析xml或properties内容，得到对于的实体类字节码字符串以及相关属性信息
	3. 使用反射机制，根据这个字符串得到某个类的Class实例
	4. 动态配置实例的属性


## 为什么Java只有值传递
按值调用(call by value)表示方法接收的是调用者提供的值，而按引用调用（call by reference)表示方法接收的是调用者提供的变量地址。一个方法可以修改传递引用所对应的变量值，而不能修改传递值调用所对应的变量值。

Java 程序设计语言总是采用按值调用。也就是说，方法得到的是所有参数值的一个拷贝，也就是说，方法不能修改传递给它的任何参数变量的内容。

- 例1:

```java
public static void main(String[] args) {
    int num1 = 10;
    int num2 = 20;

    swap(num1, num2);

    System.out.println("num1 = " + num1);
    System.out.println("num2 = " + num2);
}

public static void swap(int a, int b) {
    int temp = a;
    a = b;
    b = temp;

    System.out.println("a = " + a);
    System.out.println("b = " + b);
}

```
**结果：**
```java
a = 20
b = 10
num1 = 10
num2 = 20
```

**解析：**

swap方法中，a、b的值进行交换并不会影响num1、num2。因为a、b中的值，只是从num1、num2复制过来。实际上a、b中的值相当于num1、num2副本，副本内容无论怎么修改也不会影响到原本的值。

通过上面的例子，已经知道**一个方法不能修改一个基本数据类型的参数，而对象引用参数就不一样**。

**方法得到的是对象引用的拷贝，对象引用及其他的拷贝同时引用同一个对象**

### 总结
对象引用是按值传递的
- 一个方法不能修改一个基本数据类型的参数(即数值型或布尔型)
- 一个方法不可以改变对象参数的状态
- 一个方法不能让对象参数引用新的对象

## final关键字的使用
主要作用于三个地方：变量、方法、类

1. 对于final变量：如果是基本数据类型，数值一旦初始化后便不能更改；如果是引用类型的变量，初始化之后就不能让其指向另一个对象。
2. 对于final类，表明类不能被继承。final类中的所有成员方法都会被隐式指定为final方法。
3. 对于final方法，是把方法锁定，以防任何继承类修改它的含义。


## 异常处理

### 异常层次结构
所有异常都有一个共同祖先```Throwable```类。```Throwable```有两个重要的子类```Exception```(异常)和```Error```(错误)。```Exception```可以背程序本身处理(try-catch),```Error```无法处理。
```Exception```和```Error```都是Java异常处理的子类

- ```Exception```：本身可以处理的异常，可以通过catch捕获，可以分为受检查异常(必须处理)和不受检查异常(可以不处理)。
- ```Error```：属于程序无法处理的错误，我们没办法通过catch来捕获，比如OOM。


> **受检查异常** 如果没有被catch/throw处理的话是无法通过编译，比如IO异常。<br>
> **不受检查异常** 不处理也可以正常通过编译，比如数组越界、空指针等。

#### 异常处理总结
- ```try```：用于捕获一场。后面可以跟多个```catch```，如果没有```catch```，必须要有一个```finally```块。
- ```catch```：用于处理try捕获到的异常。
- ```finally```：无论是否捕获或处理异常，```finally```里面的语句都会被执行；当```try```或者```catch```遇到```return```语句，```finally```语句将在方法返回之前被执行。

**以下三种情况，```finally```不会被执行：**
1. 在```try```或者```finally```用了```System.exit(int)```退出程序。但是如果```System.exit(int)```在异常语句之后，```finally```还是会被执行；
2. 程序所在的线程死亡；
3. 关闭CPU。

**注意：**当try和finally都有return时候，在方法返回之前，finally语句内容将被执行，并且finally语句返回值会覆盖原始的返回值。比如

```java
    public static int f(int value) {
        try {
            return value * value;
        } finally {
            if (value == 2) {
                return 0;
            }
        }
    }

```

如果调用```f(2)```，返回值是0，因为finally返回值覆盖了try语句的返回值
